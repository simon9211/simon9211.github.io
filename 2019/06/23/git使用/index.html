git使用

发表于 2019-06-23 | 0 comments

常用

git checkout branchname 切换分支

git checkout -- <file> 当文件修改后未add，将修改重置，即忽略本次修改。使用git stash 将本地修改保存后git stash clear将修改清除 等效

git reset HEAD test 将git add test步骤回退，即将不需要add的文件回退

git stash 保存本地的修改，并且工作区重置

git stash pop 将缓存堆栈中的第一个stash删除，并将对应修改应用到当前的工作目录下。

git stash apply 将缓存堆栈中的stash多次应用到工作目录中，但并不删除stash拷贝

git stash list 查看本地保存记录列表

git stash clear清除本地保存记录列表

不小心commit了一个不应该commit的修改，但是还没有push，想撤销那个commit

a)git log 查看所有commit记录，找到该commit的上一次commit_id

b)git reset --hard commit_id

【该操作会将commit的修改的内容不会保留】

git reset --soft [commit_id] 就可以回滚到某一个commit，然后保留下修改的内容

撤销某次已经push到origin master
先撤回commit

config

b)git config --list 查看配置，默认--global,--loacl,--system

配置用户名/邮箱
git config --global user.name 'Simon'
git config --global user.email 'Simon9211@126.com'

暂存区 stash

把文件a.m更改为b.m
git mv a.m b.m
git add b.m
git rm a.m
git commit -m"change file name as b"

等效于上面
gir mv a.m b.m

git reset --hard 清除暂存区未提交的记录

查看git历史

git log 所有

`Merge: 0ad280d 14aafbe Author: xiwang wang simon9211@126.com
Date: Mon Apr 23 12:55:02 2018 +0800

aaa

commit 0ad280dd73fbc1ba950b6a30bad56950ef0524ce

git log --oneline

37ad4b0 (HEAD -> master, origin/master, origin/HEAD) aaa

git log -n2 --oneline 显示最近两次记录

git log --all 显示所有分支记录

git log --oneline --all -n4 --graph graph 图形化

git cat-file -p b28ab89aae18a7 查看文件对象内容

git cat-file -t b28ab89aae18a7 查看文件树内容

git init watch_git_objects

git checkout b28ab89aae18a7 分离头指针(回滚)

HEAD

git diff b28ab8 aae18a7 比较两次commit的差异

git diff HEAD HEAD^1 比较HEAD指向提交和上一次（父亲）提交差异。HEAD^,HEAD^1,HEAD~,HEAD~1 父亲 HEAD^1^1，HEAD^^,HEAD~2 父亲的父亲

Branch

git branch -av 查看所有分支

git branch newbranch创建新分支

git checkout -b newBranch 创建并切换至新分支

git branch -d branchname 删除分支

git branch -D branchname即使没有merge，也能删除分支

修改commit信息

git commit --amend 修改最近一次提交信息记录

git rebase -i 0ad280dd73fb 修改指定的commit信息

1、pick为不改，

2、reword表示修改,

3、squash表示将commit合并到上一次commit，但是不会将commit信息丢弃，fixup则会丢弃，需连续的commit。间隔的提交需手动将其连续

git diff --cached 比较暂存区和HEAD所含文件的差异

git reset HEAD 将暂存区恢复与HEAD一致

git reset HEAD --filename将指定文件的暂存区恢复与HEAD一致

git checkout --filename将工作区恢复成原来的样子

本地备份

--bare 不拷贝工作区域

git clone --bare /Users/xiwangwang/Documents/iOS/gitRepository/AirSandboxSwift/.git ya.git 哑协议

git clone --bare file:///Users/xiwangwang/Documents/iOS/gitRepository/AirSandboxSwift/.git zhineng.git智能协议

远端备份

merge

先pull

1、多人协作开发，修改不同文件，push前先fetch远端分支，后merge，再push

git fetch github 拉取远端分支

git merge github/feature/branch 合并代码

git push

2、多人协作开发，修改同一文件不同区域同一，能自动merge

git fetch 拉取远端分支

git merge github/feature/branch 合并代码

git push

3、多人协作开发，修改同一文件同一区域，不能自动merge

git fetch 拉取远端分支

git pull 拉取head-first代码 解决冲突

git commit -am"reslove reflects" 提交解决冲突

git merge github/feature/branch 合并代码

4、A修改文件名后 提交，B修改名字之前的文件内容
先pull后 自动将文件内容修改更新的已修改名称文件
